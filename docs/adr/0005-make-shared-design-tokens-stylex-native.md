# Make shared design tokens StyleX-native

`@repo/design-tokens` is a product-neutral, StyleX-native source package with two public layers: reusable foundations for spacing, typography, radii, shadows, and motion, plus role-based semantic variables, especially for color. Typography includes semantic family roles with neutral fallbacks, while applications own font assets and may override those roles—for example, `blog-astro` loads and selects MiSans. Raw palettes remain private, product UI consumes semantic colors, and component-specific tokens are introduced only after a repeated cross-package need. Shared components prefer intrinsic layout or container queries; viewport breakpoints, blog layout, asset loading, one-off geometry, and non-StyleX token formats stay with their consumers, while local color literals are reserved for externally owned identity, content, data, or artwork.
