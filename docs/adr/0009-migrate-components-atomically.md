# Migrate components atomically

Tailwind and StyleX may coexist temporarily while the migration is in progress, but each component is converted atomically rather than mixing both systems inside one component. The migration targets visual and behavioral parity rather than redesign, except for clear bugs discovered during conversion. Before bulk conversion, an end-to-end tracer slice must prove application-owned compilation for shared tokens, a native `.astro` component, a React component imported from `@repo/comment`, transitively imported prose CSS, development HMR, and a production build. Tailwind, its plugins, and its supporting utilities are removed only after every consuming surface has moved and the final application checks pass. Verification is a one-time migration audit using the repository's existing checks and targeted visual inspection; the migration does not add persistent screenshot tests or custom CI policy checks.
