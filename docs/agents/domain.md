# Domain Docs

This is a single-context repository.

Before exploring or changing a domain area, read:

- `CONTEXT.md` at the repository root for canonical language;
- relevant ADRs under `docs/adr/` for architectural decisions.

If either location does not exist, proceed silently. Domain-modeling workflows create files lazily when terms or decisions are resolved.

Use glossary terms as defined and avoid listed synonyms. Surface any proposed work that contradicts an ADR instead of silently overriding it.
